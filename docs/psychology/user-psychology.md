# User Psychology

## Why This Matters

Interfaces are not just visual arrangements—they are psychological experiences. Understanding how users think, feel, and behave ensures interfaces are not just usable but effective and satisfying.

## Cognitive Principles

### Mental Models

Users approach interfaces with pre-existing mental models:
- **Desktop metaphor** — Files, folders, trash can
- **Web metaphor** — Pages, links, navigation
- **Mobile metaphor** — Apps, gestures, notifications
- **Physical metaphor** — Buttons, switches, sliders

WHITE DIAMOND leverages these mental models rather than inventing new ones.

### Cognitive Load

Users have limited cognitive resources:
- **Intrinsic load** — The complexity of the task itself
- **Extraneous load** — The complexity of the interface
- **Germane load** — The effort to learn and remember

WHITE DIAMOND minimizes extraneous load so users can focus on their tasks.

### Pattern Recognition

Users recognize patterns before reading content:
- **Visual patterns** — Similar elements look similar
- **Behavioral patterns** — Similar elements behave similarly
- **Spatial patterns** — Similar elements exist in similar positions
- **Temporal patterns** — Similar transitions occur with similar timing

WHITE DIAMOND uses consistent patterns to enable rapid recognition.

## Emotional Principles

### Trust

Interfaces must build trust:
- **Consistency** — Same behavior, same result
- **Transparency** — Clear about what happens
- **Reliability** — Works as expected
- **Honesty** — Doesn't mislead or deceive

### Control

Users must feel in control:
- **Predictability** — Actions have expected results
- **Reversibility** — Actions can be undone
- **Visibility** — State is always visible
- **Feedback** — Actions produce immediate feedback

### Competence

Users must feel capable:
- **Achievable** — Tasks are completable
- **Clear** — Instructions are understandable
- **Progressive** — Complexity increases gradually
- **Supportive** — Help is available when needed

### Pleasure

Interfaces should be pleasant:
- **Aesthetic** — Visually pleasing
- **Responsive** — Feels alive and reactive
- **Delightful** — Contains moments of joy
- **Respectful** — Doesn't waste user's time

## Behavioral Principles

### Fitts's Law

The time to reach a target is proportional to:
- **Distance** — How far the target is
- **Size** — How large the target is

WHITE DIAMOND applies this by:
- Making important targets larger
- Placing important targets closer to common positions
- Using edge and corner positions for frequent actions

### Hick's Law

The time to make a decision is proportional to:
- **Number of choices** — More choices = longer time
- **Complexity of choices** — Complex choices = longer time

WHITE DIAMOND applies this by:
- Limiting choices per view
- Using progressive disclosure
- Grouping related options
- Providing defaults

### Jakob's Law

Users spend most of their time on other sites:
- **Expectation** — They expect your site to work like others
- **Convention** — They expect common patterns
- **Familiarity** — They prefer familiar designs

WHITE DIAMOND applies this by:
- Following platform conventions
- Using common patterns
- Respecting user expectations
- Not reinventing common interactions

### Miller's Law

Users can hold approximately 7 (±2) items in working memory:
- **Chunking** — Group information into chunks
- **Limiting** — Don't present too many options
- **Organizing** — Use clear hierarchy
- **Repeating** — Reinforce important information

WHITE DIAMOND applies this by:
- Limiting navigation items
- Chunking complex information
- Using clear visual hierarchy
- Repeating key information

### Pareto Principle

80% of effects come from 20% of causes:
- **Prioritize** — Focus on the most important features
- **Simplify** — Reduce complexity for common tasks
- **Optimize** — Make common tasks fast
- **Surface** — Make important features prominent

WHITE DIAMOND applies this by:
- Identifying core user tasks
- Optimizing for those tasks
- Progressive disclosure for advanced features
- Clear hierarchy of importance

## Psychological Patterns

### The Peak-End Rule

Users judge experiences by peaks and ends:
- **Peak** — The most intense moment
- **End** — The final moment

WHITE DIAMOND applies this by:
- Making completion feel rewarding
- Ensuring positive final impressions
- Creating moments of delight
- Avoiding negative endings

### Loss Aversion

Users prefer avoiding losses to acquiring gains:
- **Framing** — Present options as gains, not losses
- **Defaults** — Use sensible defaults
- **Confirmation** — Confirm before destructive actions
- **Recovery** — Make it easy to undo

WHITE DIAMOND applies this by:
- Confirming before deletion
- Using positive framing
- Preserving user work
- Offering recovery options

### Social Proof

Users look to others for guidance:
- **Reviews** — Show what others think
- **Activity** — Show what others are doing
- **Popularity** — Show what's popular
- **Testimonials** — Show success stories

WHITE DIAMOND applies this by:
- Showing relevant social proof
- Displaying user counts
- Highlighting popular options
- Showing recent activity

### Anchoring

Users rely heavily on first information:
- **First impression** — Initial appearance matters
- **First price** — Initial price sets expectations
- **First option** — First option is often chosen
- **First experience** — Initial experience sets tone

WHITE DIAMOND applies this by:
- Making first impressions positive
- Presenting options strategically
- Setting appropriate expectations
- Ensuring onboarding success
