# Prohibited Animations (Reference)

## Why This Document Exists

This reference document consolidates all information about prohibited animations in the WHITE DIAMOND system. Prohibited animations are not arbitrary restrictions—they are architectural violations. Each prohibition exists because the animation violates the fundamental physics of WHITE DIAMOND's spatial reality.

This document provides a concise reference for prohibited animation types and their correct alternatives. For detailed exploration, see the linked specification documents.

---

## Core Principle

**Every animation must answer: "What spatial relationship does this motion reveal?"**

If an animation cannot answer this question, it does not belong in WHITE DIAMOND. Animations that exist for "delight," "personality," or "engagement" are prohibited because they violate the physics of space.

---

## Prohibited Types

### No Teleportation

Objects that appear or disappear without transition violate spatial continuity.

**Violation:** Objects jump from one position to another without moving through continuous space.

**Correct alternative:** Objects must enter from outside the viewport and exit to outside the viewport through continuous motion.

### No Materialization

Objects that appear from nothing violate object permanence.

**Violation:** Objects materialize from non-existence, as if created from nothing.

**Correct alternative:** Objects must enter from outside the viewport, demonstrating that they exist in a larger space.

### No Dematerialization

Objects that disappear into nothing violate object permanence.

**Violation:** Objects dematerialize into non-existence, as if destroyed.

**Correct alternative:** Objects must exit to outside the viewport, demonstrating that they continue to exist beyond the viewport.

### No Bounce

Bounce implies organic elasticity, not architectural structure.

**Violation:** Objects bounce when they arrive, suggesting organic material rather than architectural construction.

**Correct alternative:** Objects decelerate smoothly (ease-out), demonstrating architectural mass and friction.

### No Wobble

Wobble implies instability, not permanence.

**Violation:** Objects wobble when they arrive, suggesting instability rather than permanence.

**Correct alternative:** Objects arrive with stable, predictable motion, demonstrating architectural solidity.

### No Spin (for emphasis)

Spin implies playfulness, not purpose.

**Violation:** Objects spin to draw attention, suggesting playfulness rather than purpose.

**Correct alternative:** Objects use shadow, border, or position changes to draw attention, demonstrating architectural importance.

### No Fade Without Movement

Fade without movement implies appearing from nothing, violating spatial continuity.

**Violation:** Objects fade in or out without spatial movement, as if materializing or dematerializing.

**Correct alternative:** Objects move through continuous space while fading, demonstrating spatial continuity.

### No Emphasis Without Spatial Meaning

Motion that draws attention without revealing structure is prohibited.

**Violation:** Objects pulse, glow, or animate to draw attention without revealing spatial relationships.

**Correct alternative:** Motion must reveal spatial relationships—entrance, exit, translation, or transformation.

## Permitted Types

### Entrance

Objects enter from outside the viewport. Demonstrates spatial structure.

- Slide from edge (left, right, top, bottom)
- Scale from origin (button → dropdown)
- Reveal from behind (scroll → content)

### Exit

Objects exit to outside the viewport. Demonstrates spatial structure.

- Slide to edge (left, right, top, bottom)
- Scale to origin (dropdown → button)
- Hide behind (content → scroll)

### Translation

Objects move from one position to another. Demonstrates spatial relationships.

- Move between positions
- Reorder within lists
- Reorganize within grids

### Transformation

Objects change size or shape. Demonstrates spatial constraints.

- Grow/shrink (emphasis/de-emphasis)
- Expand/collapse (detail reveal)

### State Change

Objects change visual properties to communicate state. Demonstrates object properties.

- Color change (hover, active, disabled)
- Border change (focus, error, success)
- Shadow change (elevation, depth)
