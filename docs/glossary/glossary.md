# Glossary

## Why This Matters

A glossary ensures consistent terminology across all WHITE DIAMOND documents. When everyone uses the same words to mean the same things, communication is clearer and misunderstandings are reduced.

## A

### Accessibility

The design practice of making interfaces usable by people with disabilities. Accessibility ensures interfaces work for users who are blind, deaf, motor-impaired, or cognitively different.

### Affordance

A visual cue that suggests how an element can be used. For example, a button's raised appearance suggests it can be pressed.

### Alignment

The arrangement of elements along a common line or edge. Alignment creates order and visual relationships between elements.

### Animation

The rapid display of images in sequence to create the illusion of movement. In interfaces, animation communicates spatial relationships and state changes.

## B

### Baseline

The invisible line upon which text sits. Baseline alignment ensures text appears aligned across columns.

### Border

A line that defines the edge of an element. Borders communicate separation, containment, and hierarchy.

### Boundary

The limits within which an element exists. Boundaries define where one thing ends and another begins.

## C

### Cognitive Load

The mental effort required to use an interface. High cognitive load makes interfaces difficult to use; low cognitive load makes them easy.

### Color Contrast

The difference in luminance between text and its background. Sufficient contrast ensures text is readable.

### Component

A self-contained, reusable element that encapsulates structure, style, and behavior. Components are the building blocks of interfaces.

### Container

An element that contains other elements. Containers define spatial relationships and grouping.

## D

### Default State

The normal, resting appearance of an element. The default state is how elements look when not being interacted with.

### Depth

The perceived distance of elements from the viewer. Depth is communicated through size, shadow, position, and focus.

### Disabled State

The appearance of an element that is not available for interaction. Disabled elements are visually muted and not focusable.

## E

### Easing

The rate of change of an animation over time. Easing functions control acceleration and deceleration.

### Element

A single component or part of the interface. Elements include buttons, inputs, text, images, and containers.

### Entrance Animation

An animation that shows an element appearing in the scene. Entrance animations communicate where elements come from.

### Exit Animation

An animation that shows an element leaving the scene. Exit animations communicate where elements go.

## F

### Focus

The state of an element being ready for keyboard interaction. Focused elements have visible indicators.

### Focus Indicator

A visual cue that shows which element currently has keyboard focus. Focus indicators are essential for keyboard navigation.

### Font Family

A set of fonts that share a common design. Font families include variations like regular, bold, and italic.

### Font Size

The height of text characters, typically measured in pixels or points.

### Font Weight

The thickness of text characters. Font weights include regular, medium, semibold, and bold.

## G

### Gap

The space between items in a container. Gaps create visual separation and rhythm.

### Grid

An invisible structure that organizes space. Grids provide alignment and consistency.

### Grid Column

A vertical division of a grid. Columns define where elements can be placed.

### Gutter

The space between grid columns. Gutters create separation between columns.

## H

### Hierarchy

The arrangement of elements by importance. Hierarchy guides the eye and creates visual order.

### Hover State

The appearance of an element when the cursor is over it. Hover states indicate interactivity.

## I

### Icon

A visual symbol that represents an action, object, or concept. Icons communicate meaning quickly.

### Inline Element

An element that exists within the content flow. Inline elements don't break the flow of text.

### Input

An interactive element that captures user data. Inputs include text fields, checkboxes, and selects.

## J

### Justification

The alignment of text along one edge. Left justification aligns text to the left; right justification aligns to the right.

## K

### Keyboard Accessibility

The ability to navigate and interact with an interface using only a keyboard. Keyboard accessibility is essential for users with motor impairments.

## L

### Label

Text that identifies an element. Labels describe what inputs are for and what buttons do.

### Layer

A level in the depth hierarchy. Layers organize elements from background to foreground.

### Line Height

The distance between lines of text. Line height affects readability.

### Loading State

The appearance of an element while processing. Loading states communicate that something is happening.

## M

### Margin

The space outside an element's boundaries. Margins create separation between elements.

### Mental Model

A user's internal understanding of how an interface works. Mental models are based on previous experience.

### Modal

An overlay that requires user interaction before returning to the main interface. Modals focus attention.

### Motion

The visual movement of elements. Motion communicates spatial relationships and state changes.

## N

### Navigation

Elements that help users move through the interface. Navigation includes links, menus, and tabs.

## O

### Object

Any discrete element that exists in the interface. Objects include buttons, inputs, text, and containers.

### Overflow

What happens when content exceeds its boundaries. Overflow can be hidden, scrolled, or truncated.

## P

### Padding

The space inside an element's boundaries. Padding creates space between content and borders.

### Primary Color

The main brand color, used for key actions and emphasis.

### Progressive Disclosure

Revealing information gradually, showing only what's needed at each step.

## R

### Radius

The rounding of border corners. Radius creates soft, friendly appearances.

### Readability

How easy text is to read. Readability is affected by font size, line height, line length, and contrast.

### Responsive Design

Design that adapts to different screen sizes. Responsive design ensures interfaces work on all devices.

## S

### Scale

The size relationship between elements. Scale creates hierarchy and proportion.

### Screen Reader

Assistive technology that reads interface content aloud for blind users.

### Secondary Color

A supporting brand color, used for less prominent actions.

### Shadow

A visual effect that simulates depth. Shadows communicate elevation and layering.

### Spacing

The space between elements. Spacing creates relationships, hierarchy, and rhythm.

### State

The condition of an element. States include default, hover, active, focus, disabled, loading, and error.

## T

### Touch Target

The area that responds to touch input. Touch targets must be at least 44x44 points.

### Transition

A smooth change from one state to another. Transitions make state changes feel natural.

### Typography

The art and technique of arranging text. Typography affects readability, hierarchy, and personality.

## V

### Vertical Alignment

The alignment of elements along a vertical line. Vertical alignment creates visual order.

### Viewport

The visible area of the interface. The viewport is the user's window into the interface.

### Visual Hierarchy

The arrangement of elements by visual importance. Visual hierarchy guides the eye.

## W

### Whitespace

The empty space between elements. Whitespace creates breathing room and visual clarity.

### Width

The horizontal extent of an element. Width affects layout and proportion.
