# Prohibited Animations

## Why This Matters

Prohibited animations are not arbitrary restrictions—they are architectural violations. Each prohibition exists because the animation violates the fundamental physics of WHITE DIAMOND's spatial reality. When an animation is prohibited, it's because it breaks the coherent spatial language that users rely on to understand the interface.

Understanding these prohibitions is essential for maintaining architectural integrity. Every prohibited animation has a specific architectural violation, a cognitive cost, and a correct alternative. This document explains not just what is prohibited, but why—so designers can make informed decisions about motion.

## Architectural Prohibitions

### No Teleportation

Objects that appear or disappear without transition violate spatial continuity. In WHITE DIAMOND, objects exist in continuous space—they enter from outside the viewport and exit to outside the viewport. Teleportation breaks this continuity.

**Architectural violation:** Teleportation implies that space is discontinuous—that objects can exist in one location and instantly exist in another. This contradicts the fundamental principle that the interface is a window into a vast architectural space.

**Cognitive cost:** Teleportation breaks spatial memory. Users cannot predict where objects will appear because objects don't follow spatial rules. This creates disorientation and reduces the user's sense of control.

**Correct alternative:** Objects must enter from outside the viewport and exit to outside the viewport. If an object needs to appear in a new location, it must travel there continuously—sliding, scaling, or fading in from a specific direction.

**Example:**
- **Prohibited:** A modal appears instantly in the center of the screen.
- **Correct:** A modal slides in from the right side of the viewport, demonstrating that it exists in a specific spatial location.

### No Materialization

Objects that appear from nothing violate object permanence. In WHITE DIAMOND, objects exist continuously—they don't materialize from non-existence. Materialization implies that objects are created from nothing, which contradicts the architectural reality.

**Architectural violation:** Materialization implies that space is empty and objects are created within it. This contradicts the principle that the interface is a window into a vast architectural space where objects already exist.

**Cognitive cost:** Materialization breaks object permanence. Users cannot trust that objects will persist because objects can appear and disappear without warning. This creates anxiety and reduces the user's sense of stability.

**Correct alternative:** Objects must enter from outside the viewport. If an object needs to appear, it must slide, scale, or fade in from a specific direction—demonstrating that it exists in a larger space beyond the viewport.

**Example:**
- **Prohibited:** A notification badge appears instantly on a button.
- **Correct:** A notification badge scales in from the center of the button, demonstrating that it's approaching from a distance.

### No Dematerialization

Objects that disappear into nothing violate object permanence. In WHITE DIAMOND, objects exist continuously—they don't dematerialize into non-existence. Dematerialization implies that objects are destroyed, which contradicts the architectural reality.

**Architectural violation:** Dematerialization implies that space consumes objects—that objects can cease to exist. This contradicts the principle that the interface is a window into a vast architectural space where objects persist.

**Cognitive cost:** Dematerialization breaks object permanence. Users cannot trust that objects will persist because objects can disappear without warning. This creates anxiety and reduces the user's sense of stability.

**Correct alternative:** Objects must exit to outside the viewport. If an object needs to disappear, it must slide, scale, or fade out to a specific direction—demonstrating that it continues to exist beyond the viewport.

**Example:**
- **Prohibited:** A menu disappears instantly when the user clicks outside.
- **Correct:** A menu slides out to the right, demonstrating that it continues to exist beyond the viewport.

### No Duplication

Objects that split into multiple copies violate object identity. In WHITE DIAMOND, each object has a unique identity—it doesn't duplicate or split. Duplication implies that objects can be copied, which contradicts the architectural reality.

**Architectural violation:** Duplication implies that space can create copies of objects—that object identity is not unique. This contradicts the principle that each object is a unique architectural construction.

**Cognitive cost:** Duplication breaks object identity. Users cannot track objects because objects can multiply. This creates confusion and reduces the user's sense of control.

**Correct alternative:** Objects must maintain their identity during movement. If an object needs to appear in multiple locations, it must move between locations continuously—demonstrating that it's the same object moving through space.

**Example:**
- **Prohibited:** A button splits into multiple copies when dragged.
- **Correct:** A button moves from one location to another, demonstrating that it's the same object moving through space.

## Accessibility Prohibitions

### No Vestibular-Inducing Motion

Motion that can cause vestibular disorders (dizziness, nausea, disorientation) is prohibited. This includes:

- **Full-screen parallax** — Moving entire screens in response to scroll. This violates the principle that the viewport is a fixed window into space—it implies that the entire space is moving, which is disorienting.

- **Auto-playing video backgrounds** — Constant motion behind content. This violates the principle that motion should be purposeful—background motion serves no spatial communication purpose and creates visual noise.

- **Pulsing elements** — Continuous rhythmic motion. This violates the principle that motion should be physical—continuous pulsing implies that objects are under constant force, which is physically unrealistic.

- **Spinning elements** — Continuous rotation. This violates the principle that motion should be physical—continuous rotation implies that objects are under constant torque, which is physically unrealistic.

- **Zooming transitions** — Full-screen zoom in/out effects. This violates the principle that objects should enter from outside the viewport—zooming implies that the entire space is changing scale, which is disorienting.

**Alternative:** Provide reduced motion options, or use subtle motion that doesn't affect the vestibular system.

### No Flashing Content

Content that flashes more than three times per second is prohibited. This includes:

- **Rapid color changes** — Alternating colors quickly. This violates the principle that motion should be physical—rapid color changes imply that objects are emitting energy, which is physically unrealistic.

- **Blinking elements** — Elements that appear and disappear rapidly. This violates the principle of object permanence—blinking implies that objects can materialize and dematerialize instantly.

- **Stroboscopic effects** — Repeated flashes. This violates the principle that motion should be physical—stroboscopic effects create visual artifacts that don't exist in physical reality.

**Alternative:** Use smooth transitions or single flashes.

### No Auto-Scrolling

Forcing content to scroll automatically is prohibited. This includes:

- **Auto-advancing carousels** — Automatically moving to the next slide. This violates the principle that users should control motion—auto-scrolling implies that the interface is controlling the user's attention, not the other way around.

- **Auto-scrolling feeds** — Automatically scrolling to new content. This violates the principle that users should control motion—auto-scrolling disorients users and breaks their spatial memory.

- **Parallax scrolling** — Background scrolling at different speeds. This violates the principle that the viewport is a fixed window—parallax implies that different layers of space move at different speeds, which is disorienting.

**Alternative:** Allow user-initiated scrolling.

## Cognitive Prohibitions

### No Unexpected Motion

Motion that surprises users is prohibited. This includes:

- **Unexpected direction** — Elements moving in unexpected directions. This violates the principle that motion should be consistent—unexpected direction breaks spatial predictability.

- **Unexpected speed** — Elements moving faster or slower than expected. This violates the principle that motion should be proportional—unexpected speed breaks temporal predictability.

- **Unexpected scale** — Elements changing size dramatically. This violates the principle that motion should be physical—unexpected scale implies that objects are changing mass, which is physically unrealistic.

- **Unexpected timing** — Elements appearing when not expected. This violates the principle that motion should be purposeful—unexpected timing implies that motion is decorative, not functional.

**Alternative:** Motion should match user expectations based on context.

### No Distracting Motion

Motion that draws attention away from the primary task is prohibited. This includes:

- **Background animations** — Continuous motion behind content. This violates the principle that motion should be purposeful—background motion serves no spatial communication purpose and creates visual noise.

- **Decorative animations** — Motion without purpose. This violates the principle that motion should be purposeful—decorative animation is motion without spatial communication, which is prohibited in WHITE DIAMOND.

- **Notification animations** — Overly prominent notifications. This violates the principle that motion should be proportional—overly prominent notifications draw attention away from the primary task.

- **Loading animations** — Longer than necessary loading indicators. This violates the principle that motion should be purposeful—loading animations should communicate loading progress, not decorate the waiting time.

**Alternative:** Use motion purposefully and sparingly.

### No Disorienting Motion

Motion that breaks spatial understanding is prohibited. This includes:

- **Teleportation** — Elements appearing without transition. This violates spatial continuity—objects must move through continuous space.

- **Random movement** — Elements moving without pattern. This violates the principle that motion should be consistent—random movement breaks spatial predictability.

- **Inconsistent directions** — Similar actions producing different directions. This violates the principle that motion should be consistent—directional inconsistency breaks spatial orientation.

- **Broken continuity** — Elements disappearing without exit animation. This violates spatial continuity—objects must exit to outside the viewport, not disappear into nothing.

**Alternative:** Maintain spatial continuity.

## Performance Prohibitions

### No Heavy Animations

Animations that impact performance are prohibited. This includes:

- **Many simultaneous animations** — Too many elements animating at once. This violates the principle that motion should be purposeful—many simultaneous animations create visual noise and reduce performance.

- **Complex path animations** — Elements following complex paths. This violates the principle that motion should be physical—complex paths imply that objects are following mathematical curves, not physical laws.

- **High-frequency updates** — Animations updating faster than 60fps. This violates the principle that motion should be physical—high-frequency updates create smooth motion that doesn't exist in physical reality.

- **Large element animations** — Animating very large elements. This violates the principle that motion should be proportional—large element animations draw unnecessary attention and reduce performance.

**Alternative:** Optimize animations for performance.

### No Layout-Triggering Animations

Animations that trigger layout recalculation are prohibited. This includes:

- **Animating layout properties** — Width, height, position, margin, padding. This violates the principle that motion should be physical—animating layout properties implies that objects are changing their physical dimensions, which is physically unrealistic.

- **Animating text properties** — Font size, line height, letter spacing. This violates the principle that motion should be physical—animating text properties implies that text is changing its physical properties, which is physically unrealistic.

- **Animating border properties** — Border width, border radius. This violates the principle that motion should be physical—animating border properties implies that boundaries are changing their physical properties, which is physically unrealistic.

**Alternative:** Animate only transform and opacity properties.

## Platform Prohibitions

### No Platform-Inconsistent Motion

Motion that contradicts platform conventions is prohibited. This includes:

- **iOS-style motion on Android** — Platform-specific animation patterns. This violates the principle that motion should be consistent—platform-inconsistent motion breaks user expectations.

- **Web-style motion on mobile** — Desktop-oriented animations on touch devices. This violates the principle that motion should be appropriate—web-style motion doesn't account for touch interactions.

- **Desktop-style motion on mobile** — Mouse-oriented animations on touch devices. This violates the principle that motion should be appropriate—desktop-style motion doesn't account for touch interactions.

**Alternative:** Follow platform-specific motion guidelines.

### No Device-Inappropriate Motion

Motion that doesn't suit the device is prohibited. This includes:

- **Hover effects on touch devices** — Effects that require hover state. This violates the principle that motion should be appropriate—hover effects don't work on touch devices.

- **Click effects on touch devices** — Effects that require click state. This violates the principle that motion should be appropriate—click effects don't account for touch interactions.

- **Precision animations on imprecise devices** — Animations requiring pixel precision. This violates the principle that motion should be appropriate—precision animations don't account for touch imprecision.

**Alternative:** Adapt motion to device capabilities.

## User Preference Prohibitions

### No Forced Motion

Motion that cannot be disabled is prohibited. This includes:

- **No reduced motion option** — Animations that can't be turned off. This violates the principle that users should control motion—forced motion implies that the interface is controlling the user's experience.

- **No animation preferences** — No way to adjust animation speed. This violates the principle that users should control motion—animation preferences allow users to customize their experience.

- **No motion control** — No way to pause or stop animations. This violates the principle that users should control motion—motion control allows users to manage their experience.

**Alternative:** Always respect user motion preferences.

### No Time-Consuming Motion

Motion that takes too long is prohibited. This includes:

- **Slow transitions** — Transitions longer than 600ms. This violates the principle that motion should be proportional—slow transitions imply that the transformation is more significant than it actually is.

- **Long loading animations** — Loading indicators longer than 3 seconds. This violates the principle that motion should be purposeful—long loading animations waste user time without communicating useful information.

- **Extended entrance animations** — Elements taking too long to appear. This violates the principle that motion should be proportional—extended entrance animations imply that the object is more significant than it actually is.

**Alternative:** Keep animations brief and purposeful.

## The Cognitive Cost of Prohibited Animations

Every prohibited animation has a cognitive cost—a mental burden that users must bear to understand the interface. These costs include:

- **Spatial disorientation** — Users lose their sense of where objects are in space. This happens when animations violate spatial continuity (teleportation, materialization, dematerialization).

- **Temporal confusion** — Users lose their sense of when things happen. This happens when animations violate temporal predictability (unexpected timing, inconsistent duration).

- **Object permanence anxiety** — Users worry that objects will disappear without warning. This happens when animations violate object permanence (dematerialization, blinking).

- **Control loss** — Users feel that the interface is controlling them, not the other way around. This happens when animations violate user control (auto-scrolling, forced motion, distracting motion).

- **Cognitive overload** — Users become overwhelmed by visual noise. This happens when animations violate purpose (decorative animations, background animations, heavy animations).

## What Prohibited Animations Communicate vs. What They Should Communicate

| Prohibited Animation | What It Communicates | What It Should Communicate |
|----------------------|----------------------|----------------------------|
| Teleportation | Space is discontinuous | Objects move through continuous space |
| Materialization | Objects are created from nothing | Objects exist in a larger space |
| Dematerialization | Objects are destroyed | Objects continue to exist beyond the viewport |
| Duplication | Objects can be copied | Objects have unique identity |
| Vestibular motion | Space is unstable | Space is stable and predictable |
| Flashing content | Objects emit energy | Objects have physical properties |
| Auto-scrolling | Interface controls user | User controls interface |
| Unexpected motion | Motion is arbitrary | Motion follows physical laws |
| Distracting motion | Motion is decorative | Motion is purposeful |
| Disorienting motion | Space is chaotic | Space is ordered and predictable |
